# 3_空间数据模型

# 空间数据模型
  通过一组空间数据对象对真实世界地理实体或现象的一种抽象表达和建模的结果，以支持对空间数据对象对地图显示、查询、编辑和分析。分为概念模型、逻辑模型、物理模型。
# 地理空间
在地学上，地理空间指地球表面及近地表空间，是地球上大气圈、水圈、生物圈等交互作用的区域。
# 地理空间实体
对复杂地理事务和现象进行简化抽象得到的结果，它的一个特征就是与一定的地理空间位置有关，都有一定的几何形态、分布状况以及相互关系。
# 地理空间的特征（空间实体的特征）

- 空间位置特征：在坐标系中的空间位置或几何定位，常采用地理坐标的经纬度、空间直角坐标等表示，也称几何特征包括空间实体位置、大小、形状和分布状况等。
- 属性特征：非空间特征或专题特征，与空间实体相联系、表示空间实体本身性质等数据或数量，如实体的类型、定义等
- 时间特征：空间实体随时间变化而变化
- 空间关系特征：实体一般不是独立存在的，而相互之间存在着密切的联系也就是空间关系。包括：拓扑关系、顺序关系、度量关系
# 空间数据模型的分类
人们首先对地理事务进行观测，认知其类型、特征、行为和关系，再对它进行分析、判别归类、简化、抽象和综合取舍。先对现实世界进行抽象、描述和表达，逐步得到概念模型，进而转换为逻辑模型和物理模型。
# 概念模型
地理空间中地理事物与现象对抽象概念集，是地理数据对语义解释。

- 场模型：描述空间中连续分布现象
- 对象模型：描述各种空间地物
- 网络模型：模拟现实世界中各种网络
- 时空模型：对地理对象或现象对时间特征进行建模
- 多维模型：对于三维及以上纬度对地理问题进行建模
# 对象模型（要素模型）（概念模型）
将研究的整个地理空间看成一个空域，地理现象和空间实体作为独立对象分布在该空域中。按照空间特征分成点、线、面、体4种基本对象，对象间可构成复杂对象，且互相保持连续比如拓扑关系。每个对象对应着一组相关的属性以区分各不同的对象。
# 场模型（域模型）（概念模型）
把地理空间中的现象作为连续的变量或体来看待，如：大气污染程度、地表温度、地形高度等，根据不同等应用场可表现三维或二维。
# 网络模型（概念模型）
与对象模型类似，描述不连续等地理现象，不同之处在于需要考虑通过路径相互连接多个地理现象之间联通情况。
# 时空模型（概念模型）
表达地理现象或实体等特征或相互关系随时间变化等动态过程和静态结果。在时空模型中，空间、时间和属性构成连地理现象或对象等三个基本要素。
时空模型最大的优点在于能同时处理空间纬度和时间纬度，从而实现数据历史状态重建、时空变化跟踪及发展态势预测等功能。
# 多维模型（概念模型）
数据纬度多于两个纬度等三维数据模型或者更多纬度等数据模型。通常用于表达某种地理现象或实体等属性或相互关系在特定区域不仅随着时间变化而且其变化还随着其他属性变化发生改变的问题。
# 逻辑模型
作为概念模型向物理模型转换的桥梁，根据概念模型确定的空间信息内容，以计算机能理解和处理的形式，具体表达空间实体及其关系。一般采取：矢量、栅格、矢量-栅格数据等模型
# 矢量数据模型
一种产生于计算机地图制图的数据模型，在矢量数据模型中，点实体用一对空间坐标表示，线实体用一窜坐标表示，面由边界线表示，表为首位相连的坐标串。
# 栅格数据模型
比较适用于用场模型抽象表达空间对象，采用面域或空域的枚举来直接描述空间实体。栅格可以用数字矩阵表示，地理空间坐标隐含在矩阵行列中。
# 矢量-栅格一体化模型
对地理空间实体同时按矢量数据模型和栅格数据模型来表示。面状实体边界采用矢量数据模拟描述，内部采用栅格数据模型表达。线状实体一般采用矢量数据模拟表达，线经过位置以栅格单元进行填充。点实体同时描述空间坐标及栅格单元位置。
# 镶嵌数据模拟
镶嵌数据模型采取规则或者不规则对小面块几何来逼近自然不规则对地理单元，适用于场模型抽象对地理现象。通过描述小面块对几何形态、相邻关系及面块内属性特征对变化来建立空间数据的逻辑数据模型。
# 面向对象数据模型
面向对象的方法就是以接近人类通常思维方式的思想，将客观世界的一切实体模型化为对象。每一种对象都有各自的内部状态和运动规律，不同对象之间的相互联系和相互作用就构成了各种不同的系统。
# 面向对象数据模型四种核心技术

- 分类：分相同属性结构和操纵方法
- 概括：把几个具有相同部分属性对象组合起来，形成一个更高层次、更具有一般性的超类过程。
- 联合：同一类对象中几个有相同部分属性值的对象组合起来，形成一个更高水平的几何对象的过程。
- 聚集：将几个不同类的对象组合成一个更高级的复合对象
# 空间数据类型

- 几何图形数据：各种类型的地图和实测数据，不仅反应空间实体的地理位置还反应实体的空间关系。
- 影像数据：卫星遥感、航空遥感和摄影测量
- 属性数据：实测数据、文字报告或地图中各类符号说明以及从遥感影像数据通过解译得到的信息
- 元数据：对空间数据进行推理、分析和总结得到对关于数据的数据。如：数据来源、数据权属、数据产生时间、暑假见精度、数据分辨率等
# 空间数据表示
点、线、面、体等基本图形要素
# 像元大小
像元大小决定了影像数据所表达对象等详细程度，一般用相同宽带和高度方格表示。
# 像元取值
栅格单元取值是唯一，由于收到栅格大小等限制，栅格单元中可能会出现多个地物，也就是混合像元等概念。
# 像元取值方法

- 中心点法：用位于栅格中心处地物类型决定取值。
- 面积占优法：以占矩形区域面积最大等地物类型作为栅格单元代码
- 重要性法：根据栅格内不同地物等重要性，选取最重要等地物类型作为相应栅格单元代码。如城镇、水系、交通等
- 百分比法：根据矩形区域内各地理要素中所占面积等百分比确定栅格单元取值。
# 空间关系
地理空间实体相互作用，包括拓扑、顺序、度量空间关系
# 拓扑空间关系
满足拓扑几何学原理的各空间数据相互关系，如：点与点的邻接、点与面的包含、线与面的相离、面与面的重合关系。

- 邻接关系：图形同类元素之间拓扑关系
- 关联关系：不同类元素
- 包含关系：不同类或同类不同级
- 连接关系：弧段之间
# 空间数据拓扑关系作用

- 反应实体之间逻辑结构关系
- 利用拓扑关系利于空间要素查询
- 根据拓扑关系重建地理实体
# 顺序空间关系
描述实体在地理空间上排列顺序，如实体之间前后、上下、等方位关系
# 度量空间关系
描述空间实体之间等距离远近等关系。
# 空间数据的逻辑模型的主要设计内容
逻辑模型设计的主要内容：
①地理实体特征的提取和抽象
②考虑建模对象的凢何特征
③考虑属性域：属性还具有一定的约束性条件，可以是数值范围，也可以是一个有效值的列表。
